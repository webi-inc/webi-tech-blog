## 3줄 요약
- 온체인 조사관 ZachXBT에 따르면 수백 개의 EVM 지갑이 미스터리한 공격으로 소액 암호화폐를 탈취당했으며, 각 피해자는 2,000달러 미만의 손실을 입었습니다.
- 공격 벡터는 MetaMask로 위장한 피싱 이메일로 추정되며, 12월 Trust Wallet 해킹 사건과 연관 가능성이 제기되고 있습니다.
- Trust Wallet 해킹은 Sha1-Hulud 공급망 공격으로 발생했으며, 700만 달러의 손실과 2,596개의 지갑이 침해되었습니다.

## 📌 주요 내용

### EVM 지갑 대규모 해킹 사건 발생

온체인 조사 전문가 ZachXBT가 보고한 바에 따르면, 수백 개의 이더리움 가상 머신(EVM) 체인 기반 암호화폐 지갑이 광범위한 해킹 공격을 받았습니다. 이번 EVM 지갑 해킹 사건은 각 피해자당 2,000달러 미만의 소액을 탈취하는 방식으로 진행되었으며, 정확한 공격 벡터는 아직 확인되지 않았습니다.

공격은 여러 EVM 호환 네트워크에 걸쳐 발생했으며, 이는 단일 블록체인에 국한되지 않은 광범위한 사이버 공격임을 시사합니다. 사이버보안 연구원 Vladimir S.는 MetaMask로 위장한 사기성 이메일이 공격 경로였을 가능성을 제기했습니다.

### 공격 방식과 보안 권고사항

사이버보안 제공업체 Hackless는 이번 사건을 "자동화된 광범위 네트워크 취약점 공격"으로 분석하며, 사용자들에게 스마트 컨트랙트 승인을 취소하고 지갑을 지속적으로 모니터링할 것을 권고했습니다.

Vladimir S.는 이번 EVM 지갑 해킹이 크리스마스에 발생한 Trust Wallet 해킹 사건과 연관되어 있을 가능성을 제시했습니다. 이는 암호화폐 보유자들이 자금과 민감한 정보를 보호하기 위해 온라인 안전 조치를 철저히 실행해야 한다는 점을 강조합니다.

### Trust Wallet 해킹 사건의 배경

지난 12월 25일 발생한 Trust Wallet 해킹으로 700만 달러의 손실이 발생했으며, 약 2,596개의 지갑이 침해되었습니다. Trust Wallet의 사건 보고서에 따르면, 이 해킹은 11월에 발생한 "Sha1-Hulud" 공급망 공격으로 인해 발생했을 가능성이 높습니다.

공급망 공격은 암호화폐 프로젝트들이 블록체인 애플리케이션을 구축하는 데 일반적으로 사용하는 npm 소프트웨어 패키지를 침해했습니다. 공격자는 Trust Wallet의 GitHub에서 개발자 "시크릿"을 탈취하여 브라우저 확장 프로그램의 소스 코드에 접근했습니다.

### 공격의 세부 시나리오

해커는 악성 버전의 확장 프로그램을 크롬 웹 스토어에 정상 확장 프로그램으로 위장하여 업로드했습니다. 바이낸스의 공동 창립자이자 전 CEO인 Changpeng "CZ" Zhao는 이 사건이 Trust Wallet의 소스 코드에 대한 깊은 지식을 가진 내부자에 의한 것일 수 있다는 의견에 동의했습니다.

Trust Wallet의 구글 크롬 기반 브라우저 확장 프로그램이 공격 대상이었으며, 모바일 애플리케이션은 영향을 받지 않았습니다. Trust Wallet을 소유한 바이낸스는 사용자들의 손실을 보상하기로 합의했습니다.

### 암호화폐 보안 트렌드

정부 간 블록체인 자문가 Anndy Lian은 "이런 종류의 '해킹'은 자연스럽지 않습니다. 내부자일 가능성이 높습니다"라고 언급했습니다. 이는 암호화폐 생태계에서 내부 보안 관리의 중요성을 강조합니다.

PeckShield의 보고서에 따르면 12월 암호화폐 해킹으로 인한 손실이 60% 감소했지만, 피싱 공격 생태계는 여전히 활발하게 작동하고 있습니다.

## 👨‍💻 개발자에게 미치는 영향

### 보안 개발 프로세스 강화 필요

이번 EVM 지갑 해킹 사건은 개발자들에게 공급망 보안의 중요성을 재확인시켜 줍니다. npm 패키지와 같은 외부 의존성을 사용할 때는 반드시 검증된 소스를 사용하고, 정기적인 보안 감사를 수행해야 합니다.

GitHub와 같은 코드 저장소에서 민감한 정보(API 키, 시크릿 등)를 관리할 때는 환경 변수와 시크릿 관리 도구를 적극 활용해야 하며, 절대 소스 코드에 직접 포함시키지 않아야 합니다.

### 스마트 컨트랙트 승인 메커니즘 개선

Web3 애플리케이션 개발 시 스마트 컨트랙트 승인 프로세스를 보다 세밀하게 설계해야 합니다. 사용자들이 불필요한 권한을 부여하지 않도록 최소 권한 원칙을 적용하고, 승인 내역을 쉽게 확인하고 취소할 수 있는 UI/UX를 제공해야 합니다.

### 사용자 보안 교육 강화

개발자는 사용자들이 피싱 공격을 식별할 수 있도록 명확한 보안 가이드라인을 제공해야 합니다. 특히 이메일이나 메시지를 통한 지갑 연결 요청은 항상 의심하도록 교육하고, 공식 웹사이트를 통한 접근만을 권장해야 합니다.

브라우저 확장 프로그램을 개발하는 경우, 코드 서명과 배포 프로세스를 철저히 관리하여 악성 버전이 업로드되지 않도록 해야 합니다.
